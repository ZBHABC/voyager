# 拾光旅图 / Shiguang Travel Vision —— 项目介绍文档


## 一、这个项目是干什么的

「拾光旅图」是一个 AI 驱动的旅行规划 Web 应用。你打开网页，看到一张实时高德地图，上面散落着全球目的地——瑞士因特拉肯、摩洛哥马拉喀什、大理洱海、冰岛环岛……你可以在地图上随便点一个地方，也可以直接跟 AI 旅行顾问聊天，告诉它"我想去海边、预算五千、不喜欢排队"，它会记住你的偏好，然后帮你生成一份结构化的每日行程，精细到上午去哪、下午去哪、晚上吃什么。

它不是那种只输出一段文字的 AI Demo。它把地图探索、AI 对话、行程生成、偏好记忆、图片搜索、导出功能全部串在了一个网页里，是一套能从"想去哪"走到"明天就出发"的完整工具链。

---

## 二、为什么会做这个项目

最初的想法来自 GitHub 上一个叫「智旅云图」的开源项目。那个项目用 Python + FastAPI + LangChain + ChromaDB + Vue 3 搭了一套旅行规划系统，RAG 检索增强、Cross-encoder Rerank、Redis 缓存、PDF 导出……技术架构很完整，但链路偏重——后端要跑 Python 服务，前端要 Vite 构建，还得部署向量数据库。

当时我在想：能不能把同样的能力，塞进一个更"轻"的壳子里？不需要 npm install，不需要 pip install，就一个 PHP 文件和一堆原生 JavaScript，放在任何一台支持 PHP 的服务器上就能跑。

这个念头催生了「拾光旅图」。我想做的是一个**轻量但完整**的旅行助手——不依赖框架，但功能不打折。

---

## 三、技术栈：四个文件撑起一个应用

整个项目的核心代码只有四个文件：

| 文件 | 规模 | 角色 |
|------|------|------|
| `index.php` | 26 行 | HTML 入口，注入首屏数据 |
| `api.php` | 255 行 | RESTful API 路由层，20+ 接口 |
| `data.php` | 2600+ 行 | 业务逻辑、Dify/DeepSeek 集成、SSE 流式、偏好向量库 |
| `app.js` | 3500+ 行 | 前端 SPA，7 个页面模块，原生 DOM 渲染 |
| `styles.css` | — | 全局样式，深色旅行科技感视觉系统 |

没有 React、没有 Vue、没有 Webpack。前端全靠原生 JavaScript 手写 SPA 路由和 DOM 渲染；后端用 PHP 内置服务器（`php -S`）就能启动。

但功能层面接入了大量外部服务：

- **高德地图 JSAPI v2.0**：实时交互地图、地理编码、POI 搜索、路线规划
- **DeepSeek Chat API**：主力 AI 引擎，处理对话和行程生成
- **Dify 自部署工作流**：18 节点的任务路由 Workflow + 多轮对话 Chatflow，实现结构化行程编排
- **Unsplash / Pexels / Pixabay / 腾讯万象图**：四路图片聚合搜索，自动匹配目的地照片
- **豆包（Doubao）/ OpenAI 兼容接口**：AI 引擎备选方案
- **接口盒子（搜狗/百度）**：聚合图片搜索兜底

还有一个设计上的巧思：数据库不是必需的。配置了 MySQL 就用 MySQL；没配置就自动回退到本地 JSON 文件存储。这意味着在任何一个 PHP 环境里解压就能跑——零配置起步。

## 四、核心功能一览

### 4.1 世界探索（首页）

打开就是一张高德地图，全球目的地以标记点散落。点任意一个地点，弹出毛玻璃卡片：照片、简介、预算、最佳季节、与偏好的匹配度。点击地图空白处可以"就地取点"——后端自动做地理编码，把经纬度转成结构化目的地。地图支持深色/浅色主题切换，高德 SDK 跟着一起变。

### 4.2 AI 旅行顾问

一个完整的聊天面板，背后接的是 Dify Chatflow（多轮对话工作流）。用户说"春节带爸妈去暖和的地方，预算一万，不要太累"，AI 会记住预算偏好、节奏偏好、目的地倾向——全部自动写入偏好向量库。

聊天过程是 SSE 流式推送的，用户会看到 Dify 工作流的每一"步"（理解需求 → 画像匹配 → 生成建议）变成可折叠的执行轨迹，不是傻等一个转圈。

### 4.3 行程规划

填好目的地、天数、预算、节奏，点"生成行程"。后端走三级回退：

1. **Dify Workflow**（18 节点：参数校验 → 知识库检索 → 偏好提取 → 天气分析 → 行程骨架 → 内容编排 → 润色 → 预算估算 → JSON 输出）
2. **DeepSeek Chat**（Dify 不可用时自动切换）
3. **本地规则引擎**（网络挂了也能吐出结构化的每日行程）

前端实时展示 Stepper 进度条，16 个步骤节点依次亮起。生成结果包含每日的上午/下午/晚上安排、注意事项、预算拆分，支持导出文本和日历文件（ICS）。

### 4.4 推荐好地方 & 偏好画像

用户每一条聊天记录、每一次手动输入，都会被提取成语义向量。项目内建了一个轻量向量搜索引擎——不需要 Milvus 或 pgvector，纯 PHP 实现关键词提取 + TF-IDF 向量化 + 余弦相似度计算。

推荐页会按偏好画像给目的地打分排序。如果 Dify 在线，推荐刷新同样走流式 Stepper，每一步都可视化。

偏好画像页有一个可拖拽的"神经图"——6 个维度（人文/自然/美食/摄影/购物/夜生活）以 SVG 连线呈现，用户能直观看到自己的旅行人格。

### 4.5 旅行记忆 & 愿望清单

内置 10 条示例旅行记录（厦门、西安、挪威等），每条包含日期、心情、花费、每日行程回顾。卡片以图片背景 + 毛玻璃文字展示。愿望清单支持收藏/移除，一键跳转规划。

## 五、迭代过程：从 Mock 到 Dify 工作流

这个项目不是一次写成的。下面是真实的更新时间线：

### 阶段一：搭骨架（基准版本）
- 确定了 PHP + 原生 JS 的"零依赖"架构
- 写完了 7 个页面的布局和路由
- 高德地图集成，目的地 mock 数据
- 一切 API 都由 PHP mock 返回，保证离线可用

### 阶段二：接入真实 AI（2026 年 6 月上旬）
- 接入了 DeepSeek Chat API，替换 mock 回复
- 同时兼容豆包和 OpenAI 兼容接口，做三级回退
- 偏好画像从静态数据变成动态向量库
- Unsplash + Pexels + Pixabay + 接口盒子图片搜索上线

### 阶段三：Dify 工作流集成（2026 年 6 月 20-21 日）
- 自部署 Dify 服务器，编排两个应用：
  - 「拾光旅图-任务路由工作流」：18 节点，统一处理行程生成、推荐刷新、偏好分析、目的地详情
  - 「拾光旅图-AI 顾问 Chatflow」：多轮对话，支持 conversation_id 会话记忆
- PHP 端新增 11 个 Dify 核心函数、7 个 SSE 流式函数
- 前端新增 Stepper 进度组件（180 行 CSS + 120 行 JS）
- 新增 `/api/trips/generate-stream`、`/api/ai/chat-stream` 等流式接口

### 阶段四：打磨与修复（2026 年 6 月底）
- 聊天 Stepper 折叠展开、执行轨迹 DOM 增量更新
- 地图深色/浅色主题同步切换
- 偏好记忆乱码自动过滤
- 16 项 UI 审计修复（select option 颜色、过渡动画、空状态提示等）
- 设置页重构，统一表单样式

## 六、技术亮点（答辩重点）

### 6.1 三级 AI 回退机制

```
Dify Workflow 成功 → 返回 Dify 结果（最佳质量）
    ↓ 失败
DeepSeek Chat → 返回模型结果（备选方案）
    ↓ 失败
本地规则引擎 → 返回结构化行程（永不挂掉）
```

不管外部服务怎么波动，用户永远能看到一个结构完整的结果。这不是"报个错就完事"的 demo。

### 6.2 SSE 流式 + Stepper 实时进度

传统的 AI 生成是阻塞等待——点一下按钮，等 30 秒，弹出一段文本。拾光旅图的做法是：用户点击"生成行程"后，16 个步骤节点依次亮起（提交参数 → 校验 → 知识库检索 → 偏好提取 → 骨架生成 → 内容编排 → 润色 → 校验输出），每一步都有名字、有状态。这是把 AI 工作流的"内部思考"外化成用户能理解的进度。

### 6.3 无数据库也能跑

MySQL 是优先选项，但没有 MySQL 时自动回退到 `storage/*.json`。偏好向量库同理——不用装 ChromaDB 或 Milvus，PHP 自带的能力就够跑一个轻量 TF-IDF 搜索引擎。

### 6.4 安全性设计

Dify API Key、数据库密码全部放在 `.env` 文件（已 gitignore），PHP 端做白名单过滤，公开 API 只返回 `configured: true/false`。高德地图的 `securityJsCode` 强制配置，浏览器端不暴露任何密钥。
